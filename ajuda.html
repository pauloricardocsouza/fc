1Visão geral

O que o FC Filadelfia faz e como os dados fluem entre as páginas.

O que é o FC Filadelfia

O FC Filadelfia é um sistema de gestão de fluxo de caixa projetado para trabalhar em conjunto com o ERP SIA. Ele importa os títulos a pagar e a receber, combina com lançamentos manuais e saldos bancários, e apresenta uma projeção diária consolidada.

O objetivo é ter, em uma única tela, a visão de quanto vai entrar, quanto vai sair e qual o saldo projetado de cada dia.

Como os dados chegam ao sistema

  1. Importação do SIA — na página Processamento você envia duas planilhas (contas a pagar e a receber) geradas pelo SIA. Os títulos ficam disponíveis para todos os usuários assim que são salvos.
  2. Lançamentos manuais — para despesas ou receitas que não estão no SIA (impostos, pró-labore, aportes), você cadastra manualmente em Lançamentos ou direto no Fluxo de Caixa.
  3. Saldos bancários — você registra periodicamente os saldos reais de cada conta bancária. O sistema usa esse valor como ponto de partida para projetar o saldo dos próximos dias.

Sincronização em tempo real

Todos os dados ficam no Firebase. Se um colega criar um lançamento em um computador, ele aparece automaticamente na tela de todos os outros usuários logados, sem precisar recarregar a página.

O indicador verde "Sincronizado" no canto superior direito confirma que você está conectado. Se ficar amarelo ou vermelho, a conexão com o Firebase caiu.

2Perfis de acesso

Cada usuário tem um perfil que determina o que pode fazer no sistema.

Perfil O que pode fazer O que NÃO pode fazer
Admin Tudo: importar dados, criar/editar/excluir lançamentos, gerenciar categorias, promover outros usuários, ver auditoria, excluir comentários de outros. Nada é bloqueado.
Editor Importar dados do SIA, criar/editar/excluir os próprios lançamentos, lançar saldos, gerenciar bancos, criar categorias, comentar. Ver configurações administrativas, promover usuários, excluir comentários dos outros, excluir saldos lançados.

3Fluxo de Caixa

Página principal. Mostra a projeção diária de entradas, saídas e saldo.

A grade de projeção

A tabela central mostra as datas nas colunas e os fornecedores/contas nas linhas. Cada célula mostra o valor que vence naquele dia para aquele fornecedor. A última coluna soma o total do fornecedor no período.

As três linhas fixas no topo são:

  • Contas a Receber (Total) — soma de todos os títulos a receber daquele dia
  • Contas a Pagar (Total) — soma de todos os títulos a pagar daquele dia
  • Saldo do dia — resultado (entradas menos saídas) daquele dia

Expandir e recolher

O triângulo ▸ ao lado de uma linha indica que ela pode ser expandida. Clique na linha para abrir.

  • Contas a Pagar (Total) — ao expandir, mostra todas as categorias. Clicando em uma categoria, abre os fornecedores dentro dela.
  • Contas a Receber (Total) — expande para mostrar os lançamentos manuais de recebimento (só aparece se houver algum).
  • Categorias — expandem para mostrar os fornecedores que pertencem a elas.

Filtro de período

Os campos de data no topo controlam o intervalo visível. Você pode digitar datas manualmente ou usar os botões de atalho: Hoje, 7 dias, 30 dias, 90 dias.

O botão Resetar volta ao período padrão (todos os dados disponíveis).

Ordenação

Clique no cabeçalho de qualquer coluna para ordenar por ela. Clique de novo para inverter a direção. A seta ▲ ou ▼ mostra a coluna ativa.

Você pode ordenar por nome do fornecedor, por valor em um dia específico, ou pelo total do período.

Ver detalhes dos títulos

Clique em qualquer valor dentro da grade para abrir o modal de detalhes. Ele mostra cada título individual daquele fornecedor naquele dia, com documento, valor e data. Dali você pode também excluir títulos manuais.

Dica: os valores em vermelho são negativos (saídas maiores que entradas naquele dia). Os valores em azul são positivos (superávit). Fins de semana aparecem com fundo cinza levemente diferente.

4Saldos bancários

Painel abaixo da grade. Mostra quanto você tem em cada conta agora.

Contas cadastradas

O sistema vem com 19 contas pré-cadastradas, separadas em dois grupos:

  • Contas correntes (10): ABC, ITAU, SAFRA, BRADESCO, CAIXA, C6, BANCO DO BRASIL, SANTANDER, ASA, SOFISA
  • Vinculadas / garantidas (9): VINCULADA DO SAFRA, VINCULADA C6, GARANTIDA CAIXA, VINCULADA SAFRA, VINCULADA BB, GARANTIDA SOFISA, VINCULADA SANTANDER, VINCULADA ASA, VINCULADA ABC

O grupo "correntes" são contas com saldo disponível para movimentação imediata. "Vinculadas" são valores alocados como garantia, recolhimento compulsório ou outra forma de bloqueio temporário.

Lançar saldo

Use o botão + Lançar saldo para registrar o saldo atual de uma conta. Abrirá um modal onde você escolhe a conta e informa o valor.

Cada lançamento é imutável — quando você lança um novo saldo, o anterior permanece no histórico. Isso permite consultar evolução ao longo do tempo.

Você também pode lançar saldos clicando direto no valor R$ 0,00 de cada conta no painel, editando inline.

Ver histórico

O botão Ver histórico abre um modal com todos os lançamentos de saldo já feitos, agrupados por conta. Você vê quem lançou, quando e qual o valor.

+ Nova conta

Para cadastrar uma nova conta bancária além das 19 padrão, use o botão + Nova conta. Informe o nome e escolha o tipo (corrente ou vinculada).

Atenção: saldos lançados incorretamente só podem ser excluídos por admins. Editores não podem apagar o histórico — é proposital, para preservar a auditoria.

5Lançamentos manuais

Para registrar algo que não vem do SIA.

Quando usar

Use lançamentos manuais para registrar movimentações que o ERP não controla:

  • Impostos (DAS, FGTS, INSS) que você paga fora do contas a pagar do SIA
  • Pró-labore, honorários, despesas pessoais dos sócios
  • Aportes, empréstimos, rendimentos financeiros
  • Recebimentos adiantados ou adiantamentos de fornecedor
  • Qualquer ajuste pontual no fluxo projetado

Criar um lançamento

Você pode criar lançamentos de três formas:

  1. Na página Lançamentos, pelo botão + Novo lançamento
  2. Direto na grade do Fluxo de Caixa, clicando numa célula vazia de um fornecedor
  3. Pelo atalho de teclado (ver seção de atalhos)

Campos do formulário:

  • Tipo — a pagar ou a receber
  • Entidade — nome do fornecedor/cliente (vira maiúsculas automaticamente)
  • Data de vencimento — quando o lançamento afeta o caixa
  • Valor — aceita vírgula ou ponto como decimal
  • Documento — opcional, número de nota ou referência
  • Nota — opcional, observação livre

Lançamentos recorrentes

Ao criar um lançamento, marque a caixa Recorrente para gerar múltiplos lançamentos em série. As opções são:

  • Semanal — cria um por semana a partir da data inicial
  • Quinzenal — cada 14 dias
  • Mensal — mesmo dia do mês seguinte (com ajuste inteligente para meses que não têm aquele dia, como 31 de fevereiro)
  • Anual — mesmo dia do ano seguinte

Todos os lançamentos da série ficam vinculados por um identificador comum (seriesId). Se você excluir um lançamento da série, os outros permanecem — só aquele é removido.

A data final da recorrência é obrigatória para evitar criar lançamentos infinitos por engano.

Página Lançamentos

Lista todos os lançamentos manuais cadastrados, com filtros por:

  • Tipo (pagar/receber)
  • Período
  • Texto livre (busca em entidade, documento e nota)

Cada linha tem botões para editar ou excluir. Lançamentos recorrentes mostram um badge "SÉRIE" para você saber que fazem parte de um grupo.

6Comentários

Anote observações sobre células específicas da grade.

Como funciona

Cada célula da grade (combinação de fornecedor + data) pode ter um ou mais comentários. O indicador é um pequeno ponto colorido que aparece no canto da célula quando ela tem comentário.

Para adicionar um comentário, clique na célula para abrir os detalhes e use o campo de texto na parte de baixo do modal.

Quem pode editar/excluir

Cada usuário pode editar e excluir apenas os próprios comentários. Admins podem excluir comentários de qualquer usuário.

Quando você exclui, o comentário não é apagado definitivamente — é marcado como "deletado" e sai da visualização. Isso permite recuperar se precisar investigar.

Página Comentários

Lista todos os comentários do sistema, ordenados por data. Ajuda a revisar o que foi dito recentemente ou encontrar uma nota antiga por busca de texto.

7Categorias

Organize fornecedores em grupos para ver totalizadores por categoria.

Para que servem

Categorias são usadas apenas no lado de Contas a Pagar. Permitem agrupar fornecedores por natureza (Energia, Aluguel, Impostos, Matéria-prima, etc.) e ver o total gasto em cada grupo.

Na grade do fluxo, ao expandir a linha "Contas a Pagar (Total)", você vê as categorias como sub-totalizadores. Expandindo uma categoria, você vê os fornecedores dentro dela.

Categoria nativa

Existe uma categoria especial chamada DEMAIS FORNECEDORES que não pode ser excluída. Todos os fornecedores começam nela. Conforme você organiza, vai movendo para categorias específicas.

Criar e organizar

Na página Categorias:

  1. Use + Nova categoria para criar uma nova, definindo nome e cor
  2. Selecione a categoria na lista à esquerda
  3. Na coluna "Fornecedores disponíveis", escolha os que quer adicionar e clique em → Adicionar
  4. Para remover um fornecedor, selecione na coluna "Atribuídos" e clique em ← Remover (ele volta para DEMAIS FORNECEDORES)

Excluir categoria

Ao excluir uma categoria (não-nativa), todos os fornecedores atribuídos voltam automaticamente para DEMAIS FORNECEDORES. Nenhum fornecedor é perdido.

8Dashboard

Visão agregada com indicadores e rankings.

Indicadores principais (KPIs)

  • Total a Receber — soma de todos os títulos a receber no período selecionado
  • Total a Pagar — soma de todos os títulos a pagar no período
  • Resultado projetado — diferença entre entrada e saída (positivo = superávit, negativo = déficit)
  • Dias com déficit — quantos dias do período têm saldo negativo

Presets de período

Os botões no topo aplicam períodos pré-definidos: 7 dias, 30 dias, 90 dias, Mês atual, Mês anterior, Todo o período.

Top Categorias, Fornecedores e Clientes

Tabelas que mostram quem mais contribui para as contas no período:

  • Top Categorias — soma de contas a pagar por categoria, ordenado do maior para o menor
  • Top Fornecedores — fornecedores com maiores valores a pagar
  • Top Clientes — clientes com maiores valores a receber

Distribuição por dia da semana

Gráfico de barras que mostra em quais dias da semana concentram-se as entradas e saídas. Útil para identificar padrões (ex: todo final de mês tem pico de pagamentos).

9Processamento

Importação dos dados do SIA.

O que é

A página Processamento é onde você carrega os títulos a pagar e a receber gerados pelo SIA. O sistema lê as planilhas, identifica as colunas necessárias, valida os dados e salva no Firebase.

Toda importação substitui a anterior — você não "acumula" múltiplas importações. A ideia é que você sempre importe a versão mais recente e fresca do SIA (geralmente diária).

Quais arquivos enviar

O sistema precisa de dois arquivos separados:

  1. Contas a Pagar — planilha .xlsx ou .xls com os títulos a pagar
  2. Contas a Receber — planilha .xlsx ou .xls com os títulos a receber

Você arrasta cada arquivo para o card correspondente (ou clica no card e seleciona) e eles são pré-processados. Uma prévia aparece mostrando as primeiras linhas.

Como exportar do SIA

Os dois relatórios saem do menu Financeiro do SIA, no formato Consulta Analítica por Título:

  1. Contas a Pagar: SIA → Financeiro → Contas a Pagar → Consulta Analítica por Título → Exportar para Excel (gera arquivo .xls)
  2. Contas a Receber: SIA → Financeiro → Contas a Receber → Consulta Analítica por Título → Exportar para Excel (gera arquivo .xls)

Antes de exportar, aplique os filtros:

  • Vencimento: defina o período que quer projetar (geralmente 30 a 90 dias à frente)
  • Situação (apenas no Contas a Receber): "A Receber" para excluir títulos já recebidos

Estrutura dos arquivos

Os dois relatórios têm estrutura similar — as primeiras 5 linhas são o cabeçalho institucional (nome da empresa, título, data/hora da geração, filtros aplicados). A linha 6 é o cabeçalho de colunas e a partir da linha 7 vêm os dados.

Contas a Pagar (14 colunas):

Emp. · Vencimento · Emissão · Credor · Título/Parcela · (em branco) · Atraso · Valor Parcela · Saldo a Pagar · Saldo Acumulado · Agente de Cobrança · Cheque Nº · C.C./Banco · Histórico

Contas a Receber (14 colunas):

Emp. · Vencimento · Emissão · Cliente · Título/Parcela · Atraso · Valor Parcela · Saldo Receber · Descontos · Saldo Líquido · Saldo Acumulado · Cheque Nº · C.C./Banco · Histórico

O sistema identifica as colunas automaticamente pelos nomes — não importa a posição exata, desde que o cabeçalho contenha "Vencimento", "Credor"/"Cliente", "Valor Parcela" e "Título/Parcela".

Observação sobre o formato

O SIA exporta arquivos com extensão .xls, mas o conteúdo é na verdade XLSX moderno. Isso é normal — o sistema FC Filadelfia detecta automaticamente e processa corretamente.

Processar

Depois de carregar os dois arquivos, clique em Processar e salvar. O sistema:

  1. Extrai todos os títulos das duas planilhas
  2. Valida datas e valores
  3. Substitui os dados antigos no Firebase pelos novos
  4. Arquiva a importação anterior (histórico das últimas 5 importações fica preservado para rollback)
  5. Notifica todos os usuários conectados — as outras páginas atualizam sozinhas

Histórico de importações

O sistema guarda as últimas 5 importações. Admins podem reverter para uma importação anterior em Configurações → Backup, caso a última importação tenha algum problema.

Atenção: lançamentos manuais NÃO são afetados por reimportações. Eles vivem separadamente dos títulos importados. Só os dados do SIA são substituídos.

10Configurações

Página visível apenas para administradores.

Usuários

Lista todos os usuários cadastrados. Admin pode:

  • Promover editor para admin
  • Rebaixar admin para editor ou viewer
  • Ver último login de cada um

O sistema sempre exige que haja pelo menos um admin ativo — não é possível remover o último.

Auditoria

Log cronológico das últimas ações no sistema. Cada evento mostra:

  • Quem fez a ação
  • Que tipo de ação (criou lançamento, excluiu fornecedor, mudou role, importou dados, etc.)
  • Data e hora
  • Detalhes adicionais (valor, nome da entidade afetada)

A auditoria retém 60 dias. Eventos mais antigos são removidos automaticamente.

Backup

Permite baixar um arquivo JSON com todos os dados (lançamentos manuais, categorias, bancos, saldos, comentários) para guardar fora do sistema.

O backup é feito manualmente, sob demanda. Não há backup automático.

Importações

Lista as últimas importações do SIA. Admin pode reverter para uma anterior caso a mais recente tenha problemas.

12Perguntas frequentes

Dúvidas comuns e como resolver.

Fiz uma importação e os números estão errados. Como volto?

Peça ao admin para ir em Configurações → Importações e escolher a versão anterior para restaurar. Isso reverte apenas os dados do SIA — lançamentos manuais ficam intactos.

Meu colega criou um lançamento, mas não aparece na minha tela.

Normalmente aparece em segundos sem recarregar a página. Se não aparecer, verifique o indicador de sincronização (canto superior direito). Se estiver vermelho, sua conexão com o Firebase caiu — recarregue a página.

Cadastrei um saldo errado. Como corrijo?

Você não consegue excluir saldos já lançados (proposital, para preservar histórico). Lance um novo saldo com o valor correto — ele sobrescreve o anterior nas visualizações, mas o antigo permanece no histórico.

Se realmente precisa remover o lançamento errado, peça a um admin.

Criei uma série recorrente grande por engano. Como excluir tudo?

Na página Lançamentos, filtre pelos lançamentos da série (todos aparecem com o badge SÉRIE com o mesmo identificador). Exclua um a um, ou peça ajuda ao admin para fazer em bloco.

Por que algumas categorias não aparecem na grade?

Só aparecem categorias que têm pelo menos um fornecedor com valor no período filtrado. Se nenhum fornecedor da categoria tem movimentação no intervalo, ela fica oculta.

Posso usar no celular?

Sim. O sistema é responsivo e funciona em navegadores móveis (Safari iOS, Chrome Android). A grade do fluxo precisa rolar horizontalmente em telas pequenas. As demais páginas se adaptam naturalmente.

Esqueci minha senha.

Na tela de login, clique em "Esqueci minha senha". Você receberá um email com link para redefinir (enviado pelo Firebase Authentication).

Como adicionar um novo usuário?

O administrador precisa criar o usuário no Firebase Console:

  1. Acesse console.firebase.google.com → projeto filadelfia-d0046
  2. Menu lateral → Authentication → aba Users
  3. Clique em Add user, informe email e uma senha temporária
  4. Avise o usuário do email + senha. Ele faz login e, se quiser, usa "Esqueci minha senha" para redefinir
  5. Após o primeiro login, ele aparece em Configurações → Usuários com perfil Editor (você pode mudar para Admin se necessário)
